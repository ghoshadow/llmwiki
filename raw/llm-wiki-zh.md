# LLM Wiki: AI 驱动的知识管理系统

## 概述

LLM Wiki 是一个综合性知识管理系统，利用大语言模型（LLM）来摄取、组织、查询和维护结构化 Wiki 知识库。

## 核心概念

### 摄取管道

摄取管道将原始文档转换为结构化的 Wiki 页面：

1. **源输入**：将原始文档（Markdown、文本、PDF）放入 `raw/` 目录
2. **内容分析**：LLM 分析文档结构并提取关键概念
3. **页面生成**：内容按原子化原则拆分为 Wiki 页面，并添加 frontmatter
4. **索引构建**：页面注册到 `index.md` 以便检索
5. **操作日志**：所有操作记录到 `log.md` 以供审计追踪

### Wiki 结构

- **页面**：`wiki/` 目录中的原子化 Markdown 文件，带有 YAML frontmatter
- **维基链接**：使用 `[[页面名称]]` 语法实现页面之间的双向链接
- **索引**：`index.md` 作为目录和知识图谱的入口
- **日志**：`log.md` 以时间戳记录所有操作

### AI 能力

- **智能查询**：用自然语言从 Wiki 内容中获取答案
- **内容摘要**：自动生成页面摘要
- **链接推荐**：AI 推荐页面之间相关的维基链接
- **一致性检查**：验证 frontmatter、链接和格式

## 技术架构

### MCP Server

模型上下文协议（MCP）服务器是所有 Wiki 操作的统一入口。它提供：

- **MCP Transport**：通过标准 MCP 协议进行 AI Agent 集成
- **REST API**：用于 Web UI 和外部工具集成

### Web UI

基于 Next.js 15 的应用程序，提供：

- 系统概览仪表盘
- 文档导入面板
- AI 搜索面板
- 一致性检查面板
- 索引和日志查看器
- 页面浏览器

### 文件格式

#### Frontmatter

```yaml
---
title: "页面标题"
source: "源文件.md"
created: "2024-01-15T00:00:00Z"
updated: "2024-01-15T10:30:00Z"
tags: ["概念", "教程"]
summary: "页面内容的简要描述"
---
```

#### 维基链接

兼容 Obsidian 的 `[[页面名称]]` 语法，用于内部链接，并自动追踪反向链接。

## 使用场景

1. **技术文档**：维护始终最新的活文档
2. **研究知识库**：组织研究论文和发现
3. **团队 Wiki**：AI 辅助的协作知识共享
4. **学习日志**：支持间隔重复的个人知识管理

## 最佳实践

- 保持页面原子化：一个概念一个页面
- 使用一致的 frontmatter 字段
- 利用维基链接增强可发现性
- 定期运行检查以确保完整性
- 记录所有重要操作
